# this 的工作原理

JavaScript 有一套完全不同于其它语言的对 this 的处理机制。 在五种不同的情况下 ，this 指向的各不相同

# 全局范围内

console.log(this)   // window 对象

当在全部范围内使用 this，它将会指向全局对象。

# 函数调用

function foo(){
    console.log(this)   // window 对象
}
foo()

这里 this 也会指向全局对象。

# 方法调用

let test = {
    foo: function(){
        console.log(this)   // test对象
    }
}
test.foo()

这个例子中，this 指向 test 对象。

# 调用构造函数

function foo(){
    console.log(this)   // f对象
}
let f = new foo();

如果函数倾向于和 new 关键词一块使用，则我们称这个函数是 构造函数。 在函数内部，this 指向新创建的对象。

# 显式的设置 this

function foo(a, b, c) {
    console.log(this)   // bar对象
}

var bar = {};
foo.apply(bar, [1, 2, 3])   // 数组将会被扩展，如下所示
foo.call(bar, 1, 2, 3)  // 传递到foo的参数是：a = 1, b = 2, c = 3

当使用 Function.prototype 上的 call 或者 apply 方法时，函数内的 this 将会被 显式设置为函数调用的第一个参数。

因此函数调用的规则在上例中已经不适用了，在foo 函数内 this 被设置成了 bar。

# 常见误解

尽管大部分的情况都说的过去，不过第二个规则，也就是直接调用函数时，this 指向全局对象，被认为是JavaScript语言另一个错误设计的地方，因为它从来就没有实际的用途。

let Foo = {}
Foo.method = function() {
    console.log(this)   // Foo对象
    function test() {
        console.log(this)   // window 对象
    }
    test()  // 直接调用函数，内部的this指向window对象
}
Foo.method()

# 方法的赋值表达式

另一个看起来奇怪的地方是函数别名，也就是将一个方法赋值给一个变量。

let someObject = {
    methodTest: function(){
        console.log(this)   // window 对象
    }
}
var test = someObject.methodTest
test()

上例中，test 就像一个普通的函数被调用；因此，函数内的 this 将不再被指向到 someObject 对象。 虽然 this 的晚绑定特性似乎并不友好，但这确实是基于原型继承赖以生存的土壤。

function Foo() {}
Foo.prototype.method = function() {
    console.log(this)   // bar对象
}

function Bar() {}
Bar.prototype = Foo.prototype

let bar = new Bar().method()

当 method 被调用时，this 将会指向 Bar 的实例对象 bar 。
