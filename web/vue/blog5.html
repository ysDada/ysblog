# vue3.0 API

API地址[https://composition-api.vuejs.org/zh/api.html#setup]

# setup

  1. 将会在一个组件实例被创建时，初始化props之后调用，生命周期来看，位于 beforeCreate 钩子之前
  2. 可以返回一个对象--这个对象上的属性将会被暴露给模板的渲染上下文

# 参数

接受两个参数，第一个参数 props，props 对象是响应式的，watchEffect 或 watch 会观察和响应 props 的更新

export default {
  props: {
    name: String,
  },
  setup(props) {
    watchEffect(() => {
      console.log(`name is: ` + props.name)
    })
  },
}

注意不要解构props对象，那样会失去响应

第二个参数是context，相当于2.0的this，只暴露了一些property（attrs，slots，emit）

interface Data {
  [key: string]: unknown
}

interface SetupContext {
  attrs: Data
  slots: Slots
  emit: (event: string, ...args: unknown[]) => void
}

function setup(props: Data, context: SetupContext): Data

注意不要在setup中使用this，跟2.0的不一样

# 响应式系统 API

# reactive跟ref函数

一个包装对象只有一个属性：.value。包装对象的意义就在于提供一个让我们能够在函数之间以引用的方式传递任意类型值的容器

reactive：创建一个没有包装的响应对象，同于 2.x 的 Vue.observable()，内部是通过Proxy实现

const count = ref(0)
const state = reactive({
  count,
})

console.log(state.count) // 0

state.count = 1
console.log(count.value) // 1

ref：返回一个包装对象（value reference），通常访问它需要加上.value，但是内部封装了，可以直接访问，不过特定条件洗需要加上.value，比如从 Array 或者 Map 等原生集合类中访问 ref

const arr = reactive([ref(0)])
// 这里需要 .value
console.log(arr[0].value)

const map = reactive(new Map([['foo', ref(0)]]))
// 这里需要 .value
console.log(map.get('foo').value)

# computed

跟现有2.0差不多 传入一个 getter 函数，返回一个默认不可手动修改的 ref 对象。

const count = ref(1)
const plusOne = computed(() => count.value + 1)

console.log(plusOne.value) // 2

plusOne.value++ // 错误！

或者传入一个拥有 get 和 set 函数的对象，创建一个可手动修改的计算状态。

const count = ref(1)
const plusOne = computed({
  get: () => count.value + 1,
  set: (val) => {
    count.value = val - 1
  },
})

plusOne.value = 1
console.log(count.value) // 0

# watch以及watchEffect

# watch

完全等效于this.$watch

# watchEffect

立即执行传入的一个函数，并响应式追踪其依赖，并在其依赖变更时重新运行该函数，返回一个侦听器，可以调用它停止监听

const stop = watchEffect(() => {
  /* ... */
})

// 之后
stop()
